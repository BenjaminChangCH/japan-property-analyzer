## 📝 文檔維護規則

### 新增文檔
1. 根據類型放入對應目錄
2. 更新本索引文檔
3. 在 CHANGELOG.md 中記錄

### 歸檔文檔
1. 過時文檔移至 `archive/` 目錄
2. 保留重要歷史記錄
3. 更新索引移除過時連結

### 文檔命名規範
- 使用英文大寫和底線：`DESIGN_SYSTEM.md`
- 日期格式：`YYYY-MM-DD`
- 版本格式：`v1.2.0`

---

**提示**：所有文檔都使用繁體中文撰寫，遵循專案的語言規範。
